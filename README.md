# QGROUP — каталог оборудования

Веб-приложение для компании QGROUP: публичный каталог оборудования, кабинет партнёра,
админ-панель для ведения каталога и встроенный конструктор коммерческих предложений (КП).
Интерфейс двуязычный — русский (по умолчанию) и румынский.

> Это каталог-справочник, а не интернет-магазин: складских остатков, заказов, оформления
> покупки и оплаты нет. Корзина используется только для переноса товаров в конструктор КП.

## Стек и как всё устроено

| Слой | Технология | Назначение |
|---|---|---|
| Сервер | **Node.js + Express** | маршруты, бизнес-логика |
| Шаблоны | **EJS** | страницы рендерятся на сервере (SSR), не SPA |
| База | **SQLite** (better-sqlite3) | данные: товары, пользователи, КП, контакты, логи |
| Картинки | **sharp** | конвертация загруженных фото в WebP + превью |
| Excel | **exceljs** | импорт прайса и выгрузка товаров |

Запрос проходит путь: **маршрут** (`src/routes/`) → **контроллер** (`src/controllers/`) →
**модель** (`src/models/`, работа с БД) → **EJS-шаблон** (`views/`). Вся работа с базой
собрана в `src/config/db.js` (подключение + схема) и моделях — поэтому СУБД можно сменить,
не трогая контроллеры и шаблоны.

## Роли и доступ

| Роль | Вход | Розн. цена | Партн. цена | Каталог (ред.) | Пользователи | КП |
|---|---|---|---|---|---|---|
| Посетитель | нет | ✅ | — | — | — | — |
| Партнёр | да | ✅ | ✅ | — | — | партнёрский |
| Продавец (sales) | да | ✅ | ✅ | — | — | общий (+ LED по галочке) |
| Менеджер | да | ✅ | ✅ | ✅ | только партнёры | — |
| Админ | да | ✅ | ✅ | ✅ | менеджеры + партнёры | все |

**Защита партнёрской цены.** Поле `price_partner` вырезается на сервере в единственной точке —
`serializeProduct()` в [src/models/product.js](src/models/product.js). Для посетителя оно не
попадает ни в HTML, ни в JSON-ответы (не «прячется через CSS», а физически отсутствует).

## Возможности

### Каталог
Категории в боковом меню, поиск по названию/коду, **сортировка** (по названию, по цене ↑/↓,
сначала новинки), карточки с фото. Товары «по запросу» (без цены) при сортировке по цене
всегда внизу. Язык интерфейса переключается RU/RO в шапке.

### Корзина → конструктор КП
Пользователь с доступом к КП (админ / продавец / партнёр) добавляет товары из каталога в
корзину (значок в шапке), затем кнопкой «Создать КП» переходит в конструктор — выбранные
позиции подтягиваются туда автоматически (с количеством, ценой и фото). Корзина хранится в
браузере (localStorage); цена/название при импорте берутся из актуального каталога.

### Конструктор КП (коммерческие предложения)
Перенесён из Google Apps Script и встроен в сайт; хаб — на `/kp`.
- **Три билдера:** общий (`/kp/general`), LED (`/kp/led`), партнёрский (`/kp/partner`), у
  каждого есть мобильная версия `…/mobile`. HTML лежат в [kp-builders/](kp-builders/).
- **Мост вместо Google Apps Script:** клиентский [public/js/gas-shim.js](public/js/gas-shim.js)
  эмулирует `google.script.run` и вызывает REST API `POST /kp/api/<fn>`
  ([src/controllers/kp.js](src/controllers/kp.js)).
- **Данные:** каталог КП = товары сайта (розничная цена); карточки менеджеров — из профилей
  сотрудников (имя/телефон/отдел/цвет для подписи); фото товаров попадают в КП; готовые КП
  хранятся в таблице `kps`.
- **Локализация:** интерфейс билдеров переводится на RO «на лету»
  ([public/js/kp-i18n.js](public/js/kp-i18n.js)).
- **PDF:** печать средствами браузера (Ctrl/Cmd + P → «Сохранить как PDF»).

### Админ-панель
- **Товары** — создание/редактирование, загрузка фото, мягкое скрытие.
- **Пользователи** — сотрудники и партнёры, контактные данные для подписи в КП.
- **Контактные лица** — отдельный справочник (только админ), который выводится на публичной
  странице **«Контакты»**.
- **Активность** — лента аудита: изменения товаров/пользователей, входы и действия по КП
  (создание/изменение/удаление КП, открытие конструктора, отказы доступа).
- **Экспорт в Excel** — выгрузка всех товаров или выбранных категорий (только админ).

### Импорт прайса и перевод
- **Импорт из Excel** со встроенными фотографиями (см. ниже). Идемпотентен по коду товара.
- **Машинный перевод** карточек на румынский (поля `name_ro` / `description_ro`).

## Запуск локально

Нужен **Node.js 20+** (проверено на 24). На Windows для `better-sqlite3` и `sharp`
используются готовые бинарники — компилятор C++ не требуется.

```bash
npm install
cp .env.example .env          # Windows: copy .env.example .env
npm run migrate               # создать таблицы в БД
npm run seed                  # создать администратора
npm run dev                   # запуск с автоперезагрузкой (или: npm start)
```

Открыть <http://localhost:3000>. Логин и пароль администратора задаются в `.env`
(`ADMIN_LOGIN` / `ADMIN_PASSWORD`). **Смените пароль после первого входа.**

Основные переменные `.env`: `PORT`, `DATABASE_URL` (путь к файлу SQLite),
`SESSION_SECRET` (длинная случайная строка), `ADMIN_LOGIN` / `ADMIN_PASSWORD`,
параметры обработки картинок (`IMAGE_MAX_WIDTH`, `IMAGE_QUALITY` и т. д.).

## Импорт прайса

Прайс — файл `.xlsx`, где каждый лист = категория, а фотографии встроены в книгу и привязаны
к строкам. Скрипт сам находит колонки по заголовкам (код, наименование, описание, розничная и
партнёрская цены), извлекает картинку по якорю строки, конвертирует в WebP (+ превью) и
кладёт в `public/images/products`.

```bash
npm run import-price -- data/price.xlsx                 # обычный импорт
node scripts/importPrice.js data/price.xlsx --dry-run   # пробный прогон без записи в БД
node scripts/importPrice.js data/price.xlsx --no-images # без обработки картинок
```

- **Идемпотентно по `code`:** повторный запуск обновляет товары, а не плодит дубли.
- Для большого файла (десятки МБ, сотни фото) дайте Node больше памяти:
  ```bash
  node --max-old-space-size=4096 scripts/importPrice.js data/price.xlsx
  ```

## Перевод карточек на румынский

Названия и описания импортируются на русском. Поля `name_ro` / `description_ro` заполняются
машинным переводом (бесплатный публичный эндпоинт, без ключа).

```bash
npm run translate                       # перевести все пустые RO-поля
node scripts/translate.js --field=desc  # только описания
node scripts/translate.js --limit=100   # частями
node scripts/translate.js --dry-run     # без записи в БД
```

- **Возобновляемо:** заполняет только пустые RO-поля; после обрыва запустите снова.
- Чистые артикулы (без кириллицы) не переводятся — показывается оригинал.
- Качество машинное; любую карточку можно поправить вручную в админке.

## Деплой на сервер

Типовая схема: **PM2** (держит Node-процесс) + **nginx** (reverse-proxy и отдача статики) +
**HTTPS** (Let's Encrypt).

```bash
# 1. Код и зависимости
git clone <repo> /var/www/qgroup && cd /var/www/qgroup
npm ci --omit=dev
cp .env.example .env          # задать SESSION_SECRET, ADMIN_*, NODE_ENV=production
npm run migrate && npm run seed

# 2. Процесс под PM2 (автозапуск после перезагрузки сервера)
sudo npm i -g pm2
pm2 start src/index.js --name qgroup-catalog
pm2 save && pm2 startup
```

nginx проксирует на `127.0.0.1:3000`, а статику (`/images/`, `/css/`, `/js/`) отдаёт напрямую
(быстрее и без нагрузки на Node). HTTPS ставится через `certbot --nginx`. Приложение
использует `trust proxy` и secure-cookie в авто-режиме, поэтому после перехода на HTTPS
защищённые cookie включаются сами.

> Что не хранится в git и переносится отдельно: файл базы `data/qgroup.db`, реальный `.env`
> и исходный прайс `.xlsx`. После клонирования базу можно поднять заново
> (`migrate` + `seed` + `import-price`).

## Смена базы данных

Вся работа с БД изолирована в [src/config/db.js](src/config/db.js) и [src/models/](src/models/).
Чтобы перейти на PostgreSQL/MySQL:
1. Поставить драйвер (`pg` или ORM), поднять пустую базу.
2. Переписать подключение и `initSchema()` в `db.js`; SQL почти переносимый (заменить
   `AUTOINCREMENT` → `SERIAL`/`AUTO_INCREMENT`, `datetime('now')` → `now()`).
3. Запросы в моделях используют именованные плейсхолдеры (`@param`) и обычный SQL.
4. Перенести данные: повторно запустить `import-price` на новой базе.

## Безопасность

- Пароли хранятся как **bcrypt**-хэши; сессии — httpOnly + SameSite=Lax + secure (авто за HTTPS).
- На `/login` — rate-limit (20 попыток за 15 минут с одного IP).
- Партнёрская цена недоступна публично (см. [Роли и доступ](#роли-и-доступ)).
- Вывод данных экранируется (защита от XSS).
- Удаление товара — мягкое (`is_active = 0`), данные не теряются.
- Действия по КП и админке пишутся в лог (раздел «Активность»).
